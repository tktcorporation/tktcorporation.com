# Project Structure

## Root Directory Layout
```
/workspaces/tktcorporation.com/
├── src/                    # React application source code
│   ├── pages/             # Page components
│   │   ├── Home.tsx       # Homepage with profile and links
│   │   └── Resume.tsx     # Resume page with LAPRAS integration
│   ├── components/        # Reusable React components
│   ├── data/             # JSON data files
│   │   └── experiences.json # Work experience data from LAPRAS
│   ├── types/            # TypeScript type definitions
│   ├── styles/           # CSS modules and global styles
│   ├── hooks/            # Custom React hooks
│   ├── assets/           # Static assets (images, etc.)
│   ├── App.tsx           # Main app component with routing
│   ├── main.tsx          # Application entry point
│   └── firebase.ts       # Firebase configuration
├── public/               # Static public assets
├── scripts/              # Build and utility scripts
│   ├── screenshot.ts     # PR screenshot generator
│   ├── test-display.ts   # Display testing script
│   └── update-lapras-data.ts # LAPRAS data sync script
├── tests/                # Test files
├── dist/                 # Production build output (gitignored)
└── Configuration Files
    ├── package.json      # Dependencies and scripts
    ├── tsconfig.json     # TypeScript configuration
    ├── biome.json        # Linting and formatting rules
    ├── vite.config.js    # Vite build configuration
    ├── tailwind.config.js # Tailwind CSS configuration
    ├── postcss.config.js # PostCSS configuration
    ├── docker-compose.yml # Docker configuration for CI
    ├── .mcp.json         # MCP (Model Context Protocol) config
    ├── .mise.toml        # Tool version management
    ├── .nvmrc            # Node version specification
    ├── CLAUDE.md         # Claude AI guidance document
    └── README.md         # Project documentation

## Key Directories

### /src/pages
- Contains the main page components
- Each page is a standalone route in the application

### /src/components  
- Reusable UI components used across pages
- Follow React component best practices

### /src/data
- JSON data files, primarily experiences.json
- Synced with LAPRAS API via MCP integration

### /scripts
- Utility scripts for development and deployment
- Written in TypeScript, executed with tsx

## Build Output
- `/dist` - Contains production build files
- Generated by `npm run build`
- Deployed to Firebase hosting

## CI/CD
- GitHub Actions workflow on push to master and PRs
- Uses Docker Compose for consistent environment
- Runs linting, type checking, and build steps
